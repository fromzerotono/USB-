# USB 串口继电器开发接口文档

## 1. 适用范围
- 项目：USB 串口继电器 Python 控制脚本
- 代码文件：usb_conmunicate.py
- 运行环境：Linux
- Python 版本：Python 3.x
- 依赖：pyserial

## 2. 快速开始
### 2.1 安装依赖
```bash
pip3 install pyserial
```

### 2.2 运行示例
```bash
python3 usb_conmunicate.py
```

### 2.3 默认串口参数
- 串口设备：/dev/ttyUSB0
- 波特率：9600
- 超时时间：0.5 秒

## 3. 通信协议说明
指令按 4 字节发送，格式如下：

1. 起始标识：0xA0
2. 通道地址：0x01, 0x02, ...
3. 操作码：
   - 0x00：关闭（不反馈）
   - 0x01：打开（不反馈）
4. 校验码：前 3 字节求和

校验公式：
校验码 = 0xA0 + 通道地址 + 操作码

示例：
- 打开第 1 路：A0 01 01 A2
- 关闭第 1 路：A0 01 00 A1

## 4. Python 接口定义

### 4.1 usb_init
函数原型：
```python
usb_init(serial_port="/dev/ttyUSB0", baud_rate=9600, timeout=0.5)
```

功能：
- 打开串口并返回串口对象。

参数：
- serial_port: 串口设备路径，默认 /dev/ttyUSB0。
- baud_rate: 波特率，默认 9600。
- timeout: 读写超时（秒），默认 0.5。

返回：
- serial.Serial 对象。

可能异常：
- serial.SerialException（串口不存在、权限不足、设备被占用等）。

### 4.2 usb_write
函数原型：
```python
usb_write(s, cmd)
```

功能：
- 发送 16 进制字符串命令到串口。

参数：
- s: 串口对象（serial.Serial）。
- cmd: 16 进制字符串，格式如 A0 01 01 A2。

返回：
- 无返回值。

可能异常：
- ValueError（cmd 不是合法十六进制字符串）。
- serial.SerialException（串口写入失败）。

### 4.3 relay_set
函数原型：
```python
relay_set(s, channel=1, on=True)
```

功能：
- 控制指定通道的继电器开关。

参数：
- s: 串口对象（serial.Serial）。
- channel: 通道号，默认 1。
- on: 开关状态，True 为打开，False 为关闭。

行为：
- on=True 时发送操作码 0x01，吸合继电器。
- on=False 时发送操作码 0x00，断开继电器。
- 自动计算校验码并调用 usb_write 发送。

返回：
- 无返回值。

可能异常：
- serial.SerialException（串口写入失败）。
- ValueError（内部命令构造异常时）。

### 4.4 usb_close
函数原型：
```python
usb_close(s)
```

功能：
- 安全关闭串口。

参数：
- s: 串口对象（可为 None）。

返回：
- 无返回值。

说明：
- 当 s 存在且处于打开状态时执行关闭。

## 5. 推荐调用方式
建议使用 try/finally 保证串口释放：

```python
import time
from usb_conmunicate import usb_init, relay_set, usb_close

s = usb_init("/dev/ttyUSB0", 9600, 0.5)
try:
    relay_set(s, channel=1, on=True)
    time.sleep(1)
    relay_set(s, channel=1, on=False)
finally:
    usb_close(s)
```

## 6. 时序与稳定性建议
- 连续发送命令时，命令间增加 sleep 间隔，避免命令覆盖。
- 建议开关动作间隔不小于 100ms，实际项目可按设备响应能力调大。
- 程序退出前务必关闭串口，避免下次运行占用失败。
